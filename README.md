# Chopinote-AI

> 输入一段开头，AI 为你续写完整的古典钢琴曲。

给定一首曲子的前几小节（MusicXML 格式），Chopinote-AI 以自回归方式逐 token 生成风格匹配的后续乐章，输出标准 MusicXML，可在 MuseScore、Finale、Sibelius 等打谱软件中直接打开和编辑。

---

## 系统总览

```
输入 MusicXML
    │
    ▼
┌──────────┐   ┌──────────────┐   ┌──────────────────┐
│  Token 化  │──▶│  Transformer  │──▶│  评价模块          │
│  解析乐谱   │   │  自回归生成    │   │  质量评分          │
└──────────┘   └──────┬───────┘   └────────┬─────────┘
                      │                    │
                      │◀─── 实时反馈 ──────│
                      │    (生成中调参)     │
                      ▼
                输出 MusicXML
```

四大模块：**交互模块**（CLI 入口）→ **生成模块**（Transformer 模型）→ **评价模块**（音乐性打分）→ **训练模块**（两阶段课程学习）。生成过程中评价模块实时反馈，自动调整采样参数。

## 它能做什么

```
输入:  前 8 小节巴赫众赞歌 → 输出 32 小节完整钢琴曲，风格延续
输入:  一段肖邦夜曲开头   → 输出风格匹配的即兴发展段
输入:  自己哼唱的 4 小节动机 → 输出 AI 续写的完整作品
```

## 生成模块

Decoder-only Transformer，核心设计：

```
Token 序列 → [嵌入层] → [24 层 Transformer] → [采样] → 下一个 Token
                            ↕
                     RoPE 位置编码 + 小节嵌入
```

- **音程制相对音高** — 模型学到的是到主音的距离（大三度=4，纯五度=7），而非绝对 MIDI 编号，调性泛化是原生能力
- **双重位置感知** — RoPE 编码 token 在序列中的位置，小节嵌入编码所属小节，兼顾局部装饰音和长程乐章结构

## 评价模块

四层评价体系，从基础合法性到音乐性全面打分：

```
输入 MusicXML
    │
    ▼
┌────────────────────────┐
│  合法性检查              │  音域、空小节、拍号对齐等门禁检查
└───────────┬────────────┘
            ▼
┌────────────────────────┐
│  广义评价                │  统计特征（音级/密度/节奏等）+ 和声理论规则
└───────────┬────────────┘
            ▼
┌────────────────────────┐
│  狭义评价（续写场景）    │  生成部分 vs 输入开头的风格一致性和结构衔接
└───────────┬────────────┘
            ▼
        综合评分
```

## 生成-评价反馈闭环

生成过程中实时干预，避免越写越偏：

```
┌─────────────────────────────────────────────────┐
│  A 阶段：生成前                                    │
│  分析你的开头 → 提取风格画像 → 设定生成参数初值      │
└─────────────────────┬───────────────────────────┘
                      ▼
┌─────────────────────────────────────────────────┐
│  B 阶段：生成中（每小节触发）                      │
│                                                   │
│  ┌──────────────────┐  ┌──────────────────┐      │
│  │ B1 局部层         │  │ B2 全局层         │      │
│  │ 最近几节自身是否   │  │ 生成部分 vs 开头   │      │
│  │ 流畅              │  │ 是否逐渐漂移      │      │
│  │ → 调 temperature  │  │ → 拉回关键参数    │      │
│  └──────────────────┘  └──────────────────┘      │
└─────────────────────┬───────────────────────────┘
                      ▼
┌─────────────────────────────────────────────────┐
│  C 阶段：生成后                                    │
│  全量评分 → 达标则保存，不达标则建议重试              │
└─────────────────────────────────────────────────┘
```

## 快速上手

```bash
# 续写一首曲子
chopin checkpoints/step_9000.pt input.musicxml -o output.musicxml

# 启用反馈闭环（推荐）
chopin checkpoints/step_9000.pt input.musicxml --feedback

# 生成多份变体选最好的
chopin checkpoints/step_9000.pt input.musicxml -n 5 --feedback

# 指定风格预设
chopin checkpoints/step_9000.pt input.musicxml --preset romantic
```

输出文件用任意打谱软件打开即可试听和编辑。

## 模型规格

| 项目 | 值 |
|------|-----|
| 参数量 | 1.21B |
| 架构 | Decoder-only Transformer，24 层，32 注意力头 |
| 上下文 | REMI 编码，872 种 token 类型 |
| 训练数据 | ~13.7B tokens（MIDI + MusicXML 混合） |
| 训练方式 | MIDI 预训练 → MusicXML 微调两阶段课程学习 |
| 精度 | BF16 推理，约 2.6GB 显存 |

## 训练数据来源

| 类型 | 数据集 | 说明 |
|------|--------|------|
| MIDI | MAESTRO v3.0.0 | 200+ 小时钢琴演奏录音对齐 |
| MIDI | Lakh MIDI Dataset | ~45,000 首 MIDI 歌曲 |
| MIDI | GiantMIDI-Piano | 10,854 首古典钢琴曲 |
| MIDI | POP909 / MusicNet / EMOPIA | 流行钢琴、古典、情感标注 |
| MusicXML | ASAP / ATEPP | 古典钢琴曲，含演奏法标注 |
| MusicXML | Openscore | 社区开放乐谱合集 |
| MusicXML | 内建语料库 | 肖邦全集、巴赫众赞歌、贝多芬奏鸣曲等 |
| PDMX | 流行音乐乐谱 | 含演奏法/力度/踏板/装饰音标注 |
