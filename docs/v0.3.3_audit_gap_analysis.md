# v0.3.3 审计与差距分析 — 从当前状态到世界级作曲大师

> 2026-06-05, v0.3.3 全设计完成 (opt1-opt5 + fix1), 训练未启动。
> 本文档审视当前设计的结构性不足与通向世界级的路径。

---

## 一、最直接的：模型还没训练

v0.3.x 的全套设计（SSF、Voice、Fig、Cadence、DurSat、框架分离、乐句层、三粒度功能）从**未经过一次梯度下降**。ABC Engine 做了大量推理时工作，但核心 Transformer 的参数仍然是 `nn.init.normal_(mean=0.0, std=0.02)`。

- **1.21B** 参数，对比 MuseNet (GPT-2 1.5B)、MusicLM (AudioLM 860M+)、闭源系统动辄数 B，差了一个数量级
- **seq_len=4096** 限制曲式规模，一首奏鸣曲发展部轻松超过 100 小节 (~2000 tokens)，4096 仅覆盖 4-5 分钟钢琴曲
- **从未验证过**课程训练策略、Gate 机制、DPO 闭环

---

## 二、架构层面：Transformer 不知道自己是谁

当前架构有一个根本性不对称——ABC Engine 拥有全部音乐知识，但 Transformer 只接收加性信号（SSF/Voice/Fig/Cadence/DurSat embedding），不知道：

- "你现在在奏鸣曲的呈示部，正在陈述第一主题"
- "34 小节后是全局高潮，从 20 小节开始逐步加温"
- "这个 antecedent 问句的答句应该在 8 小节后出现，用 PAC 收束"

B2 调整 temperature/density 只是采样参数，不是模型的**内部状态**。模型没有"我想写高潮了"这个意图。

**缺失**: Planning Token 层——粗粒度规划序列，在生成前插入，告诉模型"接下来 8 小节的全局意图"。

## 三、音乐理解的深度

### 3.1 没有真正的对位训练

Opt4 的 ContourBias 是 B2 层的 logit 偏置——引导模型在候选 token 中偏好反向进行。但模型从未在训练数据中学到"反向进行是好的对位"。ContourBias 是拐杖，真正的对位能力在权重里。

### 3.2 没有风格建模

当前生成"一般的四声部钢琴音乐"。Chopin vs Bach 的区别不只是织体，是**调性词汇**（半音经过音/副属和弦/那波里和弦）、**节奏语言**（rubato vs 严谨）、**装饰音语法**（花腔 vs 简洁）。

这些差异可通过**风格 embedding** 建模，但 v0.3.3 没有任何风格编码。

### 3.3 没有动机发展学习

Opt1 的 MotifTransform 是变形工具（倒影/逆行/扩展）。`select_development_action()` 是硬编码启发式。真正的动机发展需要在训练数据中看到动机如何被变形，让 Transformer 内部学会"这段材料在 20 小节后以增值形式再现"的因果链。

### 3.4 跨度建模缺失

音乐的结构层次: `Token → Bar → Phrase → Section → Piece`。当前只有 Token/Bar/Section 三层有显式表示。Phrase 层由 ABC Engine 管理但模型不知。Piece 层（全曲级）完全没有 embedding。

## 四、整体架构缺口

### 4.1 没有迭代纠错

`_stage3_generate_once()` 是单向的：生成 → C 评估 → 写入。C 说"太差了"只能调下一段参数，无法退回重写。真正的作曲是**迭代精炼**：生成 → 评估 → 定位问题 → 局部重写 → 再评估。

### 4.2 上下文长度受限

4096 tokens 对古典音乐太小。Chopin Ballade No.1 (~9分钟) 在 REMI 编码下 5000-8000 tokens。扩展方法（RoPE 插值、YaRN、sliding window）未配置。

### 4.3 只有 Decoder，没有 Encoder

所有条件信号通过 embedding 注入 hidden state，相当于 decoding 时条件化。但音乐生成的核心挑战是**结构规划和内容生成的分离**。Encoder-decoder 自然地做这件事，decoder-only + 横向注入容易在长距离上丢失结构信息。

### 4.4 曲式模板太少且固定

当前 `FORM_TEMPLATES` 只有 sonata/binary/rondo/theme_variations/free。缺少：ternary (ABA)、arch (ABCBA)、fugue、minuet-trio、theme and variations with N sections 等。且模板是静态的——不随音乐实际发展调整。

## 五、推理时与训练时的鸿沟

| | 训练时 | 推理时 (ABC Engine) |
|---|---|---|
| **框架 token** | 数据中自然出现 | A1 预插入 |
| **终止式** | 数据中自然出现 | A1 规划 + cadence_embedding |
| **织体** | 数据中自然出现 | A1 预插 FigV |
| **温度调度** | 无 | B2 动态调参 |
| **动机引导** | 无 | B2 MotifTransform + bias |
| **对位偏置** | 无 | B2 ContourBias |
| **情感参数** | 无 | B2 affect_bias |

推理时有大量 ABC Engine 注入的信号在训练时**不存在对应**。模型在训练中从未见过 `ContourBias` 的 logit 调制——它学到了"平行三度是可以的"，但不知道为什么。训练-推理 gap 可能导致推理时模型对抗偏置信号。

**缓解方向**: 训练时采样循环注入同样的偏置（teacher forcing + bias injection），让模型在训练中适应。

---

## 六、评估体系的局限

当前 C 层规则评估覆盖了：
- 语法合法性 (B1 禁令)
- 统计特性 (密度/休止比/音域/力度)
- 对位合法性 (平行五八度/声部交错)
- 段落一致性 (KL/pitch class)
- 乐句质量 (终止式强度/连贯性/呼吸)

**缺失**:
- **旋律记忆性**: 一段旋律是否"可记住"（这是主观的，但可近似为重复模式×变化度）
- **和声趣味**: 和声进行是否在"可预测"和"意外"之间找到平衡
- **结构说服力**: 再现部的回归是否有"回家"的感觉
- **情感传达**: 生成的音乐是否真的传达了目标情感

## 七、从「能写」到「世界级」的距离

| 维度 | 当前 v0.3.3 | 世界级 |
|------|-----------|--------|
| **音乐知识** | SSF和声+终止式+织体+对位 | 配器、赋格、十二音、爵士和声 |
| **风格能力** | 无风格编码 | 跨风格生成（巴洛克→现代） |
| **结构感知** | 段级 (SectionPlan) | 全曲级（发展逻辑不只模板） |
| **训练数据** | 1.62M 文件，钢琴为主 | 多元乐器、多风格、有标注 |
| **参数规模** | 1.21B | 7-70B |
| **生成长度** | 4096 tokens (~5min) | 16384+ (完整奏鸣曲/交响乐章) |
| **生成速度** | 自回归逐token | 推测解码/并行生成 |
| **评估体系** | C 层规则评分 | 规则+ML+人工全闭环 |
| **迭代能力** | 一次生成 | 多轮精炼（重写/修补/变奏） |

## 八、下一步

最合理的顺序:

1. **预处理收尾** — SSF (含 beat_fields) + Figuration + Function 标注 + split
2. **启动 Phase 1 训练** — L1+L2 纯钢琴短曲, gate: empty_bar<1%, overflow=0
3. **首版 checkpoint 评估** — 用 Phase 1 模型跑生成，验证 ABC Engine 推理时效果
4. **根据输出迭代** — 决定 opt3-opt5 哪个需要调，发现训练未覆盖的新问题
5. **Phase 2 → Phase 3** — 逐步扩展数据分布
6. **DPO 闭环验证** — 在 Phase 2 中段启动自动 DPO

最核心的差距不是某个功能的缺失，而是**训练反馈循环还没跑起来**。规则提供骨架，但血肉必须在数百万 step 的训练中从数据里长出来。
